# Product

## Register

product

## Users

面向需要阅读、比较和修改本地 Markdown 的写作者、开发者、研究者与知识工作者。他们通常在桌面浏览器中处理私有资料，希望同时查看多篇文档，又不愿把内容上传到第三方服务器。

## Product Purpose

在浏览器内打开本地 Markdown 文件或文件夹，用可停靠窗格并列阅读与编辑。文档内容始终留在本地；支持原文件写回的浏览器直接保存，其余浏览器使用下载副本降级。成功意味着用户可以快速组织多个视图、可靠恢复草稿，并始终理解当前文件权限和保存结果。

## Brand Personality

冷静、可信、利落。语言直接说明能力和限制，不卖弄技术，不用夸张营销语气。界面应让内容成为视觉中心，让工具在操作过程中退到背景。

## Anti-references

- 不做完整 IDE 式重型外壳，不复制 VS Code 的活动栏和密集状态栏。
- 不使用 AI 紫渐变、霓虹发光、装饰性玻璃拟态或无意义动画。
- 不用卡片堆叠掩盖层级，也不把空状态写成营销落地页。
- 不隐藏浏览器权限差异，不把下载副本假装成原文件保存。
- 不收集文档、使用数据或遥测。

## Design Principles

- 内容优先：文件抽屉临时出现，主要面积始终留给文档。
- 本地即真实：清楚区分原文件、恢复草稿和下载副本。
- 熟悉胜过新奇：沿用编辑器、分栏、保存与冲突处理的标准心智模型。
- 渐进增强：浏览器能力不同，但核心阅读和编辑流程保持一致。
- 状态透明：未保存、权限、冲突、恢复和错误都在发生位置说明。

## Accessibility & Inclusion

以 WCAG 2.2 AA 为基线。所有核心操作可通过键盘完成，焦点清晰可见，正文与控件达到对比度要求，抽屉和对话框具备正确语义。尊重系统明暗主题与减少动态效果设置；小屏设备使用单窗格结构，避免强迫用户操作不可用的拖放界面。
